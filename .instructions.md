# Copilot CLI Instructions

You are operating inside an autonomous task runner repository.

## Goal
Implement only the requested task while preserving cross-platform behavior.

## Required environment assumptions
- AGENT_CLI can be either copilot or claude.
- PROJECTS_ROOT may be used in tasks.md paths.
- Services may run under systemd (Linux) or Task Scheduler (Windows).

## Task execution context
- Primary runtime files:
  - pc-server/src/orchestrator.ts
  - pc-server/src/index.ts
  - bot/src/index.ts
  - tasks.md
- Platform scripts:
  - scripts/linux/*
  - scripts/windows/*

## Behavioral constraints
- Do not introduce hardcoded personal paths.
- Do not remove support for either OS.
- Keep task parser backward-compatible with:
  - ## project:
  - path:
  - context:
  - - [ ] ... | priority: high|medium|low
- Keep HTTP API stable:
  - POST /run
  - POST /stop
  - GET /status

## Safety constraints
- Prefer explicit checks for missing files/paths/env vars.
- Log actionable errors when configuration is missing.
- Avoid destructive git operations.

## Validation
- After code edits, run local builds where possible:
  - cd pc-server && npm run build
  - cd bot && npm run build
- If build cannot run due to machine constraints (for example no space), report the blocker clearly.
